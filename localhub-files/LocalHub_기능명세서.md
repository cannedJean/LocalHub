# LocalHub 기능 명세서

## 1. 문서 개요

본 문서는 LocalHub 프로젝트의 기능 범위, 상세 기능, API, 데이터, 보안, 배포 및 완료 기준을 정의한다.

- 프로젝트명: LocalHub
- 서비스 형태: 대전·충청권 지역 정보 기반 익명 커뮤니티 웹 서비스
- 프론트엔드: Vue.js 3 SPA
- 백엔드: FastAPI REST API
- ORM: SQLAlchemy
- 데이터베이스: SQLite
- 프론트 배포: Netlify
- 백엔드 배포: Render
- 대상 권역: 대전·충청권
- 주요 사용자: 지역 주민, 관광객
- 인증 방식: 회원가입·로그인 없음
- 게시글 수정·삭제 권한: 작성 시 등록한 비밀번호 검증

---

## 2. 프로젝트 목적

공공데이터에 분산된 대전·충청권의 관광지, 음식점, 축제 등 지역 정보를 한 곳에서 조회하고, 지역 주민과 관광객이 회원가입 없이 정보를 공유할 수 있는 익명 지역 커뮤니티를 구축한다.

---

## 3. 대상 사용자

| 사용자 | 주요 목적 |
|---|---|
| 관광객 | 관광지, 맛집, 축제, 날씨, 위치 정보 조회 |
| 지역 주민 | 지역 게시글 작성 및 지역 정보 공유 |
| 익명 사용자 | 회원가입 없이 게시글 작성·조회·수정·삭제 |
| 모바일 사용자 | 스마트폰 브라우저에서 반응형 웹 이용 |

---

## 4. 범위 정의

### 4.1 Must Have

- 대전·충청권 단일 권역 서비스
- 제공 JSON 기반 지역정보 조회
- 데이터 출처·라이선스 문서화
- 익명 게시판 CRUD
- 비밀번호 기반 수정·삭제
- 게시글 카테고리
- 게시글 검색과 페이지네이션
- SQLite 저장
- `POST /api/chat`
- JSON 및 게시글 기반 챗봇
- 데스크톱 플로팅 챗봇
- 모바일 대응 챗봇
- Vue 3 SPA
- FastAPI REST API
- SQLAlchemy ORM
- 환경변수 보안 관리
- Netlify·Render 배포
- 기능 명세서, WBS, 발표자료, DB 파일 제출

### 4.2 Should Have

- Leaflet 기반 지도 시각화
- 관광지·음식점 마커
- 콘텐츠 유형 및 도시 필터
- 날씨 정보 연동
- 여행 적합도 표시
- 로딩·빈 상태·오류 상태
- 챗봇 추천 질문

### 4.3 Could Have

- 축제 캘린더
- 게시글 조회수
- 링크 복사
- 간단 통계
- 챗봇 대화 초기화

### 4.4 Won't Have

- 회원가입·로그인
- JWT·OAuth
- 다권역 전환
- 댓글
- 좋아요
- 북마크
- 태그
- 이미지 업로드
- 관리자 페이지
- 경로 안내
- 실시간 알림
- 다국어
- 카카오톡 공유
- 모바일 네이티브 앱
- Android·iOS·Flutter·React Native

> 비밀번호는 교육 목적 요구사항에 따라 평문 저장하되, API 응답과 로그에는 포함하지 않는다.

- 비밀번호는 API 응답에 포함하지 않는다.

# 9. JSON 데이터 구조

## 9.1 최상위 구조

```json
{
  "region": "대전_충청권",
  "contentType": "관광지",
  "contentTypeId": 12,
  "total": 335,
  "items": []
}
```

## 9.2 주요 필드

| 필드 | 설명 |
|---|---|
| contentid | 콘텐츠 고유 ID |
| contenttypeid | 콘텐츠 유형 ID |
| title | 장소명 |
| addr1 | 기본 주소 |
| addr2 | 상세 주소 |
| tel | 전화번호 |
| mapx | 경도 |
| mapy | 위도 |
| firstimage | 대표 이미지 |
| firstimage2 | 썸네일 |
| cpyrhtDivCd | 저작권 구분 |
| createdtime | 최초 등록 시각 |
| modifiedtime | 최종 수정 시각 |

## 9.3 처리 규칙

- `mapx`, `mapy`는 문자열이므로 숫자로 변환한다.
- 유효하지 않은 좌표는 지도 마커에서 제외한다.
- 이미지가 없는 경우 대체 이미지를 사용한다.
- 주소가 없어도 화면이 깨지지 않도록 처리한다.
- `contentid`는 문자열로 유지한다.
- 제공 원본 JSON은 수정하지 않는다.
- 누락 데이터를 임의로 생성하지 않는다.
