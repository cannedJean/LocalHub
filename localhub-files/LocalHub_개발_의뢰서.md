# 공공데이터 기반 지역 정보 공유 커뮤니티(LocalHub) 서비스 개발 의뢰 건

- **발신:** 지역 정보 공유 커뮤니티 개발팀
- **작성 시기:** 2026. 7.

---

## I. 개요

- **수신:** 외부 개발·협력사 담당자 귀하
- **발신:** 지역 정보 공유 커뮤니티 개발팀
- **문의:** 개발팀 (담당자: ○○ ○ / 연락처: ○○○-○○○○-○○○○)

---

## II. 요청 내용

1. 귀사의 무궁한 발전을 기원함.

2. 당사에서는 전국 5개 권역(서울, 대전/충청, 구미/경북, 광주/전라, 부산)의 공공데이터를 사전 수집·가공하여 JSON 파일로 제공하며, 귀사는 이 수집 데이터 중 1곳을 선정하여 지역 주민 및 관광객이 정보를 자유롭게 공유·소비할 수 있는 익명 커뮤니티 플랫폼(이하 `'LocalHub'`)을 구축하여야 함.

   동 서비스는 빠른 초기 렌더링에 적합한 Vue.js, Python 비동기 처리 및 응답 속도가 우수한 FastAPI, 별도 DB 서버 없이 파일 기반으로 구동되어 배포 환경의 복잡도를 최소화하는 SQLite를 기술 스택으로 채택함.

   단, 제공 JSON 외 추가 데이터 수집이 필요한 경우 반드시 해당 데이터의 사용 범위(라이선스·공공누리 유형 등)를 사전 확인하고 사용 가능 여부를 판단한 후 활용하여야 함.

   서비스 개발에 필요한 API 키는 당사가 발급·제공하며, 개발 과정에서 발생하는 외부 서비스 비용은 기존 제공 키의 범위 내에서 운영하고 정해진 예산을 초과할 수 없음.

3. 이에 따라, 귀사에서는 상기 취지를 참조하시어 아래 III항에 명시된 개발 범위 전반에 대한 구현 방향 및 수행 가능 범위를 검토하시고, 실무 적용이 가능한 방향으로 제안하여 주시기 바람. 납기는 **2026년 7월 16일(목) 오후 3시까지**이며, 동 기한 내 기본 요구사항(IV항 참조)을 충족한 결과물 및 발표 자료, 지정 산출물을 모두 **SSAFY Git Lab**에 제출하여야 함. 개발 착수 이후 기능 요구사항의 변경은 없으므로, 귀사에서는 제안 범위 내에서 일정을 구성하여 주시기 바람.

## III. 개발 요청 범위(세부)

### 1. 제공 데이터 활용 및 추가 수집

가. 당사가 사전 수집·가공하여 제공하는 JSON 파일을 기반으로 프론트엔드 및 백엔드 데이터 연동 구현 (직접 공공 API 호출 불필요)

나. 제공 JSON 외 추가 데이터가 필요한 경우, 반드시 해당 데이터의 라이선스 및 공공누리 유형(1~4유형)을 사전 확인하고 상업적·서비스 활용 가능 여부를 검토한 후 수집·사용하여야 함

다. 활용 데이터 목록(출처·라이선스·수집일 포함)을 기능 명세서 내 별도 항목으로 정리하여 산출물에 포함

### 2. 커뮤니티 기능(CRUD) 구현 - 필수

가. 별도 회원가입·로그인 없는 익명 커뮤니티로 구현하며, 사용자 인증 및 권한 체계는 적용하지 않음

나. 게시글 작성 시 제목·내용과 함께 수정용 비밀번호를 함께 등록하며, 수정·삭제 요청 시 등록된 비밀번호와의 일치 여부로만 권한을 확인함. 비밀번호는 암호화 없이 평문으로 저장·비교하며, 이는 교육 목적의 의도된 설계임

다. 선정한 1개 권역의 카테고리 게시판 - 목록 조회, 상세 조회, 작성, 수정, 삭제 API(FastAPI REST) 및 Vue.js 화면 구현

라. 커뮤니티 데이터는 반드시 DB에 저장되어 관리될 수 있어야 함

### 3. 챗봇 기능 구현 - 필수

가. FastAPI `POST /api/chat` 엔드포인트 개발(OpenAI API 활용 가능)

- 제공 JSON 데이터 기반 자연어 지역 정보 질의응답 처리

나. 주요 질의 유형(예: 권역별 관광지 추천, 축제 일정, 모범음식점 위치, 커뮤니티 게시글 검색, …)에 응답 기능

다. 채팅 UI 컴포넌트(예: 대화 히스토리 유지, 모바일 대응, 플로팅) 개발

### 4. 프론트엔드 구현 - 필수

가. 프론트엔드 프레임워크는 Vue.js 3을 사용하며, SPA 구조로 구성

### 5. 백엔드 구현(FastAPI + SQLite) - 필수

가. FastAPI 기반 REST API 서버 구축, SQLAlchemy ORM 적용 SQLite 스키마 설계

- 예: `posts`, `comments`, `locations`, …

나. API 키, DB 경로, 외부 서비스 인증 정보 등 민감 정보는 반드시 환경변수(`.env`)로 관리하며, 소스코드 및 GitHub 저장소에 직접 기재하는 것을 절대 금지함.

- `.env` 파일은 `.gitignore`에 등록하여 버전 관리 대상에서 반드시 제외

### 6. 배포 환경 구성 - 필수

가. 프론트엔드 배포 서비스 - Netlify

나. 백엔드 배포 서비스 - Render

다. 배포 완료 후 각 서비스 URL 동작 확인 및 산출물에 포함

### 7. 개발 환경 및 AI 도구 정책

가. 귀사가 요청한 개발 환경인 **VSCode Copilot + OpenAI API Key** 조합의 사용만 허용하며, 해당 환경 기준으로 개발 및 납품하여야 함

나. **Claude Code, Codex, Cursor, Antigravity 등 기타 AI 코딩 도구는 보안상 사유로 사용 불가**이며, 위반 사항 발견 시 계약 조건에 따라 처리함

## V. 기타 및 회신 요청

4. 상기 사항은 서비스 개발 착수를 위한 내부 검토 및 유관 협력사 협의의 참고자료로 활용하고자 하는바, 귀사에서는 검토 결과를 가능한 범위 내에서 정리하여 회신 바람. 제안 내용은 단일 안에 한정하지 아니하고 복수 대안 형태로 제시하여도 무방하며, 추후 논의 과정에서 보완·수정될 수 있음을 전제로 하여 주시기 바람.

5. 세부 기능 항목 및 데이터 연동 범위는 협의 과정에서 조정될 수 있으나, 개발 착수 이후 기능 추가 및 범위 변경은 원칙적으로 수용하지 아니함. 이에 귀사에서는 착수 전 검토를 충분히 거쳐 논의 가능한 수준의 자료를 사전 준비하여 주시기 바람.

6. 다음 미팅 시까지 기술 검토 의견 및 개발 일정 초안을 지참하여 참석 바람.
